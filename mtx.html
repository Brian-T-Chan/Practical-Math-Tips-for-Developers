Matrices


Image by Nina Garman from Pixabay

Matrices can be thought of as being two dimensional arrays of numbers, where each inner array has the same number of elements. The number of inner arrays is the number of rows, and the number of numbers in each of the inner arrays is the number of columns. For instance, the matrix M below has three rows and 4 columns.

var M = [

[34, 2, 99, 7],

[2, 4, 3, 3],


[3, 4, -5, 6]];

The matrix M is usually written as

\begin{pmatrix} 34 & 2 & 99 & 7\\ 2 & 4 & 3 & 3\\ 3 & 4 & -5 & 6\\ \end{pmatrix}

or as

\begin{bmatrix} 34 & 2 & 99 & 7\\ 2 & 4 & 3 & 3\\ 3 & 4 & -5 & 6\\ \end{bmatrix}

The numbers in a matrix are called the entries. This running example starts with defining some matrices.

start of running example

We will use the following variables in this running example to demonstrate some concepts.


var M = [[34, 2, 99, 7],
        [2, 4, 3, 3],
        [3, 4, -5, 6]];

var N = [[4, 2, 99, 70],
        [-2, -4, 3, 3],
        [13, 14, -5, 6]];

var P = [[444, -2, 990, 444],
        [2, 4, 6, 8],
        [1, 2, 3, 4]];

var Q = [[1, 2, 3],
        [4, 4, 5],
        [-3, 4, -2],
        [-10, -10, -10]];

var R = [[1, 2],
        [3, 4],
        [5, 6]];

var S = [[1, 2, 3],
        [4, 5, 6],
        [7, 8, 9]];

var T = [[3, 4, 1],
        [4, 6, 5],
        [7, 7, 7]];

Matrix Addition

Addition of matrices is done entry-wise when the matrices have the same number of rows and columns. The output is another matrix.

this function describes matrix addition 

var matrixAdd = function(A, B) {
  const numRows = A.length;
  const numColumns = A[0].length;
  let sum = [];
  for (let i = 0; i < numRows; i++) {
    sum[i] = [];
    for (let j = 0; j < numColumns; j++)
      sum[i][j] = A[i][j] + B[i][j];
  }
  return sum;
}

The parameters, A and B, are matrices that have the same number of rows and the same number of columns.

running example continued

As M and N have the same number of columns, I will set

var matrixSum1 = matrixAdd(M, N);

Then matrixSum1 is the following matrix.

[[38, 4, 198, 77],

[0,0,6,6],


[16,18,-10,12]]

The sum, matrixSum1, is usually written as \[\textbf{M + N}.\] Note that

console.log(matrixAdd(M, N));

and

console.log(matrixAdd(N, M));

will both display the following.

[[38, 4, 198, 77],

[0,0,6,6],


[16,18,-10,12]]

In math symbols, \[\textbf{M + N = N + M}.\]

running example continued

Also, the matrix P has three rows and four columns, so \(\textbf{N} + \textbf{P}\) makes sense, \(\textbf{M} + \textbf{P}\) makes sense, and \((\textbf{M} + \textbf{N}) + \textbf{P}\) makes sense. Note that

console.log(matrixAdd(M, matrixAdd(N, P)));

and

console.log(matrixAdd(matrixAdd(M, N), P));

will both display the following

[[482, 2, 1188, 521],

[2, 4, 12, 14],

[17, 20, -7, 16]]

In math symbols, \[\textbf{M + (N+P) = (M+N) + P}.\] So I can write \[\textbf{M + N + P}\] instead of \[\textbf{M + (N + P)}\] or \[\textbf{(M + N) + P}.\]

In math symbols, \[\textbf{M + (N+P)}\]\[\textbf{= (M+N) + P}.\] So I can write \[\textbf{M + N + P}\] instead of \[\textbf{M + (N + P)}\] or \[\textbf{(M + N) + P}.\]

Matrix Multiplication

Unlike matrix addition, matrix multiplication is not done entry-wise but done in a different way instead. Multiplication of matrices involves matching the rows of the left matrix with the columns of the right matrix, and the output is another matrix. It is done when the number of columns in the left matrix equals to the number of rows in the right matrix.

this function describes matrix multiplication 

var matrixMultiply = function(A, B) {
  const numRows = A.length;
  const numColumns = B[0].length;
  const matchLen = B.length;
  let product = [];
  for (let i = 0; i < numRows; i++) {
    product[i] = [];
    for (let j = 0; j < numColumns; j++) {
      product[i][j] = 0;
      for (let s = 0; s < matchLen; s++)
        product[i][j] += A[i][s] * B[s][j];
    }
  }
  return product;
}

The parameters A and B are matrices where the number of columns of A equals to the number of rows of B.

running example continued

The number of columns of M is 4 and the number of rows of Q is 4. So we can set

var matrixProduct1 = matrixMultiply(M, Q);

Typing

console.log(matrixProduct1);

displays the following

[[-325, 402, -156],

[-21, 2, -10],


[-26, -58, -21]]

The matrix matrixProduct1 is written as \(\textbf{MQ}\).

running example continued

The number of columns of Q is 3 and the number of rows of R is 3. So we can set

var matrixProduct2 = matrixMultiply(Q, R);

Typing

console.log(matrixProduct2);

displays the following

[[22, 28],

[41, 54],


[-1, -2],

[-90, -120]]

The matrix matrixProduct2 is written as \(\textbf{QR}\).

running example continued

About the matrices S and T. The number of rows of the matrix S is the same as the number of columns of S. For this reason, we call S a square matrix. Similarly, T is a square matrix. In addition, the number of columns of S equals to the number of rows of T, and the number of columns of T equals to the number of rows of S. So we can consider the matrix product \(\textbf{ST}\) and the matrix product \(\textbf{TS}\). Typing

console.log(matrixMultiply(S, T));

displays

[[32, 37, 32],

[74, 88, 71],


[116, 139, 110]]

and typing

console.log(matrixMultiply(T, S));

displays

[[26, 34, 42],

[63, 78, 93],


[84, 105, 126]]

So \(\textbf{ST} \neq \textbf{TS}\). This example shows that matrix multiplication is different from multiplying numbers.

running example continued

The matrix R has 3 rows, so as the matrix Q has 3 columns, the matrix product \(\textbf{QR}\) makes sense. In fact, as Q has 4 rows and as R has 2 columns, \(\textbf{QR}\) is a matrix with 4 rows and 2 columns.

Since M has 4 columns and Q has 4 rows, \(\textbf{MQ}\) makes sense. In fact, \(\textbf{MQ}\) is a matrix with 3 rows and 3 columns because M has 3 rows and Q has 3 columns.

Set

var matrixProduct3 = console.log(matrixMultiply(M,

matrixMultiply(Q, R)));

This makes sense as M has 4 columns and as \(\textbf{QR}\) has 4 rows. Next, set

var matrixProduct4 = console.log(matrixMultiply(

matrixMultiply(M, Q), R));

This makes sense as \(\textbf{MQ}\) has 3 columns and as R has 3 rows. Note that

console.log(matrixProduct3);

and

console.log(matrixProduct4);

will both display the following

[[101, 22],

[-65, -94],


[-305, -410]]

In math symbols: \[\textbf{M(QR) = (MQ)R}\]

So I can write \(\textbf{MQR}\) instead of \(\textbf{M(QR)}\) or \(\textbf{(MQ)R}\).
